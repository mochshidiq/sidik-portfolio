## How to publish on GitHub Pages
1. Create a new public repository, for example `sidik-portfolio`.
2. Upload all files and folders from this package to the repository root.
3. Make sure `index.html` is in the repository root.
4. Go to repository `Settings` → `Pages`.
5. Under `Build and deployment`, choose `Deploy from a branch`.
6. Select branch `main` and folder `/root`.
7. Save and wait for GitHub Pages to publish the site.

## Important privacy note
Do not upload confidential company data, internal screenshots, tax data, production URLs, database content, or anything under NDA.
Use public screenshots, anonymized screenshots, generated mockups, or abstract diagrams for sensitive projects.
